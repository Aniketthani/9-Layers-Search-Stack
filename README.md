# Adverse Keyword Search — Insurance Submission Analysis

## Project Structure

```
adverse_keyword_search/
├── app.py                          # Streamlit UI + orchestrator
├── pipeline.py                     # Main pipeline — ties all modules together
├── generate_samples.py             # Creates sample test documents
├── requirements.txt
├── .env.example
│
├── config/
│   └── settings.py                 # All config + env vars
│
├── modules/
│   ├── document_ingestion.py       # PDF, DOCX, XLSX, email, ACORD
│   ├── normalizer.py               # Regex + linguistic normalization
│   ├── exact_matcher.py            # Aho-Corasick token matching
│   ├── semantic_matcher.py         # Sentence embeddings + ChromaDB
│   ├── negation_detector.py        # Affirmed vs negated language
│   ├── section_weighter.py         # Section-aware confidence adjustment
│   ├── cooccurrence_scorer.py      # Cross-category risk scoring
│   ├── llm_interpreter.py          # Groq + OpenAI interpretation
│   └── audit_trail.py              # Append-only run logging
│
└── data/
    ├── keywords/
    │   └── keyword_dictionary.json  # Replace with your own
    ├── sample_documents/            # Generated by generate_samples.py
    └── vectordb/                    # ChromaDB persistent store
```

## Setup

```bash
# 1. Clone / copy project
cd adverse_keyword_search

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set up environment
cp .env.example .env
# Edit .env and add your GROQ_API_KEY and/or OPENAI_API_KEY

# 5. Generate sample documents
python generate_samples.py

# 6. Run the app
streamlit run app.py
```

## Swapping the Keyword Dictionary

Replace `data/keywords/keyword_dictionary.json` with your own file in this format:

```json
{
  "Category Name": ["keyword1", "keyword2", "keyword3"],
  "Another Category": ["kw1", "kw2"]
}
```

Or upload it directly in the Streamlit sidebar without replacing the file.

## LLM Providers

- **Groq** (default): Fast, free tier available. Uses `llama3-70b-8192`.
- **OpenAI**: Uses `gpt-4o-mini`. Switch in sidebar or set `llm_provider=openai` in settings.

Both can be toggled in the Streamlit sidebar at runtime.

## Supported Document Types

| Format | Notes |
|--------|-------|
| PDF | Full text + table extraction, page-level metadata |
| DOCX | Paragraphs + tables, section heading detection |
| XLSX | All sheets, table-to-text conversion |
| .eml | Email body parsing |
| .msg | Outlook message format |
| .html / .htm | ACORD form HTML, table-aware |
| .txt | Plain text |

## API-Ready Structure

Each module is a standalone class/function. To expose as FastAPI endpoints later:

```python
from pipeline import AnalysisPipeline, load_keywords

pipeline = AnalysisPipeline(load_keywords())
result = pipeline.run("path/to/document.pdf")
```
